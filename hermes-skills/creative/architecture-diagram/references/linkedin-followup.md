# LinkedIn follow-up post notes

Session note for using the architecture-diagram skill to support a second, later LinkedIn post on the same topic.

## What worked
- Keep the diagram as a **decision aid / system map**, not an explainer of basics.
- Make the visual and caption do different jobs: the image should show structure and tradeoffs; the caption should add the short takeaway and the question.
- For a follow-up post on the same topic, **change the angle** so it feels distinct from the morning post (e.g. from "what it is" to "how to choose / why it matters").
- Favor a **mobile-first composition** with one clear focal cluster, generous whitespace, and no crowded right-side stack.
- Verify the rendered HTML in a browser before export/publish.

## Practical output pattern
- Save the source as `./.hermes/cache/<topic>-architecture.html`.
- Export the visual as a PNG alongside it.
- Keep the diagram offline-friendly and self-contained so the final image can be reused in LinkedIn workflows.

## Reminder
This note is session-specific guidance for social follow-up posts, not a general replacement for the main architecture-diagram design system.
